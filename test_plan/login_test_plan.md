# Test Plan – Login Module

## Objective
Validate the login functionality to ensure only authorized users can access the system.

## Scope
- Username & password validation
- Error messaging
- UI elements

## Out of Scope
- Performance & load testing
- Backend DB validation

## Test Strategy
- Manual execution
- Positive & Negative scenarios
- Boundary value testing

## Environment
- OS: Windows 10
- Browser: Chrome (latest)

## Entry Criteria
- Application deployed
- Test data available

## Exit Criteria
- All test cases executed
- No critical defects open

## Risks
- Unstable environment
- Incomplete requirements
